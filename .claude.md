# .claude.md

## Project Intent
本仓库用于“运行态反馈驱动代码优化”任务交付，重点是：
- 可执行
- 可审计
- 可交接

## Response Norms
- 语言：中文优先，术语可中英混排。
- 风格：直接、可执行、少空话。
- 每次回复必须包含：
  - 当前阶段
  - 已完成动作
  - 下一步动作
  - 风险/阻塞（若存在）

## Progress Logging Protocol
- 专用文件：`WORK_PROGRESS.md`
- 每个关键动作后必须追加记录，字段至少包括：
  - Timestamp（`yyyy-MM-dd HH:mm:ss K`）
  - Stage
  - Actions
  - Files Reviewed
  - Files Changed
  - Review Checklist

## File Review Checklist (Minimum)
- 需求覆盖是否完整。
- 术语是否一致。
- 结构是否可读。
- 链接是否有效。
- 是否包含风险声明（非法律意见）。
- 是否附带可复现命令。

## Git Practice
- 分支默认：`main`
- 提交信息建议：`type(scope): summary`
- 禁止破坏性回退（如 `git reset --hard`）
- 推送前必须完成一次自查

## Handover Essentials
新同事接手时优先阅读：
1. `README.md`
2. `gptdeepsearch2_9.md`
3. `WORK_PROGRESS.md`
4. `docs/REVIEW_CHECKLIST.md`
