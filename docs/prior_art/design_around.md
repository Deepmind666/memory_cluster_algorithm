# Design-Around Strategy

## 方案 A：冲突保真优先（推荐）
- 主权利要求主线：
  - MemoryFragment 异构结构
  - 增量聚类
  - 冲突显式标记
  - 偏好向量控制压缩与降级
  - backrefs 可逆还原
- 优点：与“仅摘要”方案区分明显，技术效果路径清晰。
- 缺点：实现复杂度略高。

## 方案 B：层次索引优先
- 主线：簇级压缩 + 二级簇聚类 + 主题树索引
- 优点：强调检索吞吐提升，易写性能指标。
- 缺点：与分层记忆方案邻近，需要加强冲突处理细节。

## 方案 C：偏好策略优先
- 主线：类别/来源/时效三维偏好向量驱动保留与细节预算
- 优点：可解释、配置化强。
- 缺点：若只写偏好摘要，容易撞可控摘要专利。

## 建议取舍
- 优先采用方案 A，方案 B 作为从属实施例，方案 C 作为参数化增强点。
- 写权利要求时，至少把“冲突显式 + backrefs 可逆”与“偏好向量控制”组合进独立权利要求。
