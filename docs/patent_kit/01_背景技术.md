# 01 背景技术

在多 Agent 协同任务中，规划代理、写作代理、验证代理会持续产生中间结论、工具输出、日志与修订意见。随着任务轮次增加，共享记忆池会出现以下问题：

1. 冗余增长：同一事实被多个代理重复表达。
2. 版本不一致：不同代理对同一参数给出冲突值。
3. 上下文膨胀：下游注入上下文时 token 预算持续上升。
4. 证据断裂：摘要后无法快速回溯原始来源。

现有方案通常采用最近窗口保留或滚动摘要。该类方案在单会话场景可降低长度，但在多 Agent 共享写入场景中，仍难以同时满足“去重、冲突显式、偏好保留、可追溯”四项要求。

因此，需要一种面向多 Agent 共享记忆池的语义聚类压缩技术方案，在保证可追溯性的同时降低存储与检索开销。
