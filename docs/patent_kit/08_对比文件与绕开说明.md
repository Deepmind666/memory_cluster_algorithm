## 二、最接近现有技术-区别特征-技术效果三联表

| 最接近现有技术 | 区别特征（本方案） | 对应技术效果（已实现/可测） |
|---|---|---|
| US20110238408A1（语义聚类） | 在聚类后保留结构化冲突记录，不做冲突值静默覆盖 | 冲突可回溯、可审计；冲突簇率与冲突优先级可量化 |
| US12008332B1（偏好摘要） | 偏好向量不只作用于摘要展示，还进入保留强度决策与预算控制（ARB） | 在同等存储约束下提高关键证据保真（`detail_budget_avg` 可测） |
| US20240289863A1（共享记忆） | 增加冲突兼容门控（DMG），避免语义近似但槽位冲突的误合并 | 混合冲突簇减少，`merges_blocked_by_guard` 可观测 |
| ConversationSummaryBufferMemory（滚动摘要） | 增加 `backrefs` 可逆回溯与冲突证据图（CEG） | 不仅保留摘要，还可按证据链回放原始碎片 |

## 三、差异化叙事
1. 对比文件多聚焦“聚类”或“摘要”单点，本方案主张“聚类 + 冲突显式 + 偏好向量 + 可逆回溯”耦合闭环。
2. 对比文件中偏好常用于摘要表现层，本方案偏好参与写入、压缩、降级、检索全过程控制。
3. 对比文件中共享记忆通常强调跨代理访问，本方案强调共享写入后的治理与一致性维护。

## 四、本轮新增差异化特征（R-009）
1. 冲突证据图（CEG）
- 将冲突从“列表记录”提升为“节点-边-优先级”的可计算结构。

2. 自适应预算（ARB）
- 摘要预算不再固定，而由冲突密度、来源熵和时效共同决定。

3. 双通道门控（DMG）
- 簇合并不仅看语义相似度，还要求冲突兼容度达标。

上述特征对应的工程证据见：
- `scripts/run_ablation.py`
- `outputs/ablation_metrics.json`
- `docs/eval/ablation_report_cn.md`

## 五、建议的权利要求收束策略
- 独立权利要求锁定四个以上耦合点，避免抽象化写法。
- 从属权利要求分别限定：冲突结构字段、时效降级规则、二阶段检索策略、版本化索引。
- 增补“上界剪枝”从属权利要求（与实现保持一致）。
- 实施例中给出可量化指标，支撑技术效果。
