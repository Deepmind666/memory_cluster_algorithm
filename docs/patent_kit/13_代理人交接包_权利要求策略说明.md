# 13 代理人交接包权利要求策略说明

## 1. 目标
在不偏离现有技术交底材料的前提下，形成“核心主张稳定 + 可选实施例保留”的权利要求结构，提升授权通过率与后续抗无效能力。

## 2. 建议结构（供代理人改写）
1. 独立权利要求（方法）：保留完整流程链路
2. 独立权利要求（系统/装置）：模块化映射方法步骤
3. 独立权利要求（存储介质）：程序实现路径
4. 从属权利要求：围绕 CEG/ARB/DMG/语义精度展开技术细化

## 3. 建议主防线
- 主防线（核心）：
  - 方法主权项（总流程）
  - CEG 细化从属项
  - ARB 细化从属项
  - DMG 细化从属项
  - 语义作用域与回指细化从属项
- 次防线（可选）：
  - Prune
  - Candidate
  - ANN

## 4. 现有草案映射（以 06 为准）
- 核心优先：权利要求 1、4-11
- 可选保留：权利要求 12-14

## 5. 建议代理人重点打磨点
1. 独立权利要求中“步骤间因果关系”要写清（不是并列罗列）。
2. 参数或阈值不写死具体数值，优先写“可配置阈值/权重”。
3. 避免纯效果性语言，尽量写“技术手段 + 处理对象 + 处理结果”。
4. 从属项中补足可替换实现，防止单一点实现过窄。
5. 术语统一：记忆碎片、语义簇、冲突证据图、预算系数、双通道门控、回指解析。

## 6. 容易被质疑的点（需提前防）
1. “只是规则拼接”质疑：需强调冲突治理链路中的协同关系，而非单模块孤立。
2. “常规聚类显而易见”质疑：需突出多 Agent 冲突记忆场景与冲突兼容门控机制。
3. “支持性不足”质疑：确保说明书实施例覆盖权利要求关键要件。

## 7. 不建议作为主战场的内容
- Candidate/ANN 的性能提升叙事不作为主授权论据，只保留为可选实施例。
- 原因：不同场景下存在负加速或收益不稳定，易被审查意见针对。

## 8. 交接口令（给代理人）
“本案核心请围绕 CEG/ARB/DMG/语义精度四条主线写强，12-14 作为备选实施例保留，不以性能加速作为主创造性依据。”
