# Review Checklist

版本：v1.1  
最后更新：2026-02-09

## 使用方法
- 使用时机：每次准备提交前。
- 执行人：实现人自查 + 评审人复查。
- 规则：逐项勾选，不满足项必须给出修复计划或阻塞说明。

## A. 需求一致性
- [ ] 用户目标是否明确映射到交付物。
- [ ] 每条硬性要求是否有落地文件。
- [ ] 是否保留可交接信息。

## B. 技术质量
- [ ] 方案流程可执行（非纯概念）。
- [ ] 输入、输出、日志路径定义清晰。
- [ ] 关键步骤有失败处理策略。
- [ ] 优化策略与监测指标一一对应。

## C. 文档质量
- [ ] 标题层级规范。
- [ ] 术语统一（如“运行态反馈”“闭环优化”）。
- [ ] 引用有来源链接。
- [ ] 结论与证据可追踪。

## D. 合规与风险
- [ ] 包含“非法律意见”声明。
- [ ] 包含先行技术重叠风险段落。
- [ ] 未出现“绝对新颖性/必然授权”表述。

## E. 工程可维护性
- [ ] `WORK_PROGRESS.md` 连续更新。
- [ ] 脚本有参数说明。
- [ ] README 可指导新同事 10 分钟内上手。
